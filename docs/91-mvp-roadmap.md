# MVP Roadmap

Three phases, paced by the only calendar that matters: **the fire season**. Phase 1 must be live and drilled before the next season opens. The phases layer deliberately: Phase 1 saves lives with information, Phase 2 organizes the response around it, Phase 3 makes the platform accountable and permanent.

---

## Phase 1 — Life-safety core *(live before the next fire season)*

**Promise:** a person in a threatened village can, with no account and three taps: call for professional help, see the verified fire picture, be warned to evacuate, know where to go and which road not to take, ask for help, and be findable by family.

| Ships | Why now |
|---|---|
| Emergency Contacts | Day-one feature; works before everything and after everything fails |
| Live Wildfire Map + status banner | The shared picture; list-view fallback included from day one |
| Incident Reporting (🔥 + 🛣️ types) | Feeds the map; the verification ladder ships here or nowhere |
| Emergency Alerts (all levels, two-person RED rule) | The evacuation machine, with templates recorded in AR/Kabyle/FR |
| Emergency Requests + coordinator triage queue | The 🆘 button and the queue that catches it |
| Road Closures | Segment statuses with freshness decay |
| Shelter Finder (coordinator-updated statuses) | "Where do we go" — manager self-service comes in Phase 2 |
| Missing Persons + "I Am Safe" | The post-evacuation search, privacy-screened from day one |
| Frequently Needed Information (first 5 decks, audio) | Pre-cached life-safety knowledge; cheap, permanent |

**Also required in Phase 1 (not software):** Local Coordinators appointed and trained in the pilot wilayas (start with the highest-risk: Tizi Ouzou, Béjaïa, Bouira, Jijel…); a national Moderator desk staffed for the season; every emergency number re-verified with its authority.

**Phase 1 exit test:** a full simulated village evacuation (the doc 30 drill, run for real) — alert issued in under 2 minutes, relayed, shelter found, requests triaged, zero dead-ends.

---

## Phase 2 — Coordination layer *(built during and immediately after the first season)*

**Promise:** the response organizes itself on the platform — volunteers become missions, shelters run themselves, needs become deliveries, medical cases reach the right door.

| Ships | Builds on |
|---|---|
| Volunteer Management (registry + verification, surge variant) | Roles from Phase 1's coordinator network |
| Volunteer Assignments + Safety Check-in & SOS | The registry; requests convert to missions |
| Shelter Operations (counter, check-in registry, needs, staffing) | The finder; check-ins feed reunification |
| Family Reunification matcher | Missing persons × shelter check-ins × I-Am-Safe |
| Resource Management (chips + sector grid) | Shelter ops |
| Supply Requests / needs board (full lifecycle) | Resource chips; the logistics heart |
| Water & Food Distribution points | The needs board; volunteer missions staff them |
| In-Kind Donations (pledges, drop-offs, NOT-needed banner) | The needs board's public door |
| Medical Coordination (queue, facility board, 💊 lines, 🩺 roster) | Requests, needs board, volunteer registry |
| Safe Routes | Road closures data, now proven fresh enough to route on |
| Community Updates | The coordinator network is now dense enough to feed it |
| Local Emergency Centers directory | Coordinators maintain it |
| Misinformation Flagging + "Check a claim" | The ladder now has enough public items to defend |
| Livestock lane (🐄 request type + guide decks) | Near-zero cost riders on requests and guides |

**Phase 2 exit test:** one real supply need travels the full loop — posted at a shelter, claimed by an NGO, delivered, confirmed — and one real mission musters, checks in every volunteer, and checks every one out.

---

## Phase 3 — Accountability & permanence *(between seasons)*

**Promise:** the platform survives the news cycle: losses are counted, recovery is watched, money is verifiable, and next season starts pre-loaded instead of from zero.

| Ships | Why last |
|---|---|
| Damage Reports (with the assisted/desk filing path) | Post-crisis by nature — but ready *before* season one ends |
| Recovery Tracking boards | Fed by the damage register |
| Financial Donations directory + transparency feeds | Accreditation process has matured through Phase 2 |
| Public Statistics Dashboard | The numbers now exist and are worth showing |
| Fuel Distribution (coordinator-gated) | Needs-board rails, narrow use |
| Vulnerable Persons Registry | The flagship calm-season campaign |
| Relay Captains (role + tooling) | Appointment/training can begin in Phase 2; tooling lands here |
| Pre-Season Preparedness Mode + readiness cards + drills | The annual heartbeat; runs every year thereafter |
| Season Archive | After the first full season, by definition |

**Phase 3 exit test:** the pre-season readiness card is green in every pilot commune before the next season opens — registry fresh, shelters re-verified, captains active, drill timed.

---

## The loop, thereafter

Each season repeats the rhythm: **prepare (Phase 3 machinery) → respond (Phases 1–2 machinery) → account and recover (Phase 3 machinery) → prepare again**, one wilaya cohort wider each cycle, until NAJDA is simply how Algeria answers fire.
